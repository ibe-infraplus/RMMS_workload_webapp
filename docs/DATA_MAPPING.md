# Data Mapping Documentation (Data Dictionary)

เอกสารฉบับนี้อธิบายที่มาของตัวแปรต่างๆ (Variables) ในระบบคำนวณงบประมาณ Workload Cost ว่ามีการดึงข้อมูลมาจากไฟล์ Excel ไหน ชีตอะไร และคอลัมน์อะไรบ้าง เพื่อความสะดวกในการดูแลรักษาข้อมูลและพัฒนาต่อยอดโดยนักพัฒนาภายนอก (External Developers)

ไฟล์ข้อมูลหลัก (Data Sources) ทั้งหมดจะถูกเก็บไว้ที่โฟลเดอร์ `data/` ภายในโปรเจกต์

---

## 1. ข้อมูลพื้นฐานและปริมาณงานตั้งต้น (Baseline Quantities)
ข้อมูลเหล่านี้ถูกดึงมาจากไฟล์ `data/operating_distances.xlsx` และ `data/group_data_final.xlsx` 

### 1.1 ไฟล์ `group_data_final.xlsx`
ไฟล์นี้มีหลายชีตทำหน้าที่เก็บทั้งข้อมูลโครงสร้างพื้นฐาน ปริมาณงาน Asset และ Policy ของแต่ละแขวงทางหลวง

| ชีตใน Excel (Sheet) | ความหมายและตัวแปรหลัก |
|---------------------|----------------------|
| **group_data_final** | เก็บข้อมูลโครงสร้างพื้นฐาน เช่น `dept3`, `division_name`, `district_name`, `length_to2` (ระยะทางแบบ 2 ช่องจราจร) ฯลฯ สำหรับใช้คำนวณ Condition Score และ Pavement Workload |
| **group_asset** | เก็บปริมาณงานหมวดสินทรัพย์ เช่น `ศาลาทางหลวง`, `สะพานลอยคนเดินข้าม` (จำนวนแห่ง) |
| **input** | ข้อมูลเสริมเช่น `veh_total` (AADT รวม), ค่าเฉลี่ย IRI, ความลาดชัน ซึ่งโหลดเข้ามาเป็น Prefix `input_` |
| **ราคาต้นทุน (Asset) / ราคาค่าบำรุง** | เก็บข้อมูลราคากลาง (Unit Cost) สำหรับนำไปใช้คำนวณงบประมาณในรายการต่างๆ |

> **หมายเหตุ:** ข้อมูลนโยบาย (POLICY) เช่น `ไฟฟ้าแสงสว่าง`, `สัญญาณไฟจราจร` จะถูกดึงมาจากคอลัมน์ในไฟล์นี้เช่นกัน โดยคอลัมน์เหล่านั้นถูก Merge รวมเพื่อส่งต่อไปที่ระบบประมวลผล ข้อมูลหลักของระบบรวมถึง Unit Cost จะรวมศูนย์อยู่ในไฟล์นี้ทั้งหมด

### 1.2 ไฟล์ `operating_distances.xlsx`
ใช้เก็บข้อมูลระยะทางดำเนินการรวมของแต่ละแขวง (เพื่อนำมาอ้างอิงและตรวจสอบ ไม่ได้ผูกกับ Unit Cost โดยตรง) 

| ตัวแปรใน Code (Variable) | ชื่อคอลัมน์ใน Excel | ความหมาย/คำอธิบาย |
|-------------------------|--------------------|-------------------|
| `operating_distance` | `total_distance_km` | ระยะทางดำเนินการทั้งหมดในเขตพื้นที่รับผิดชอบของแขวง (กม.) |

## 2. ข้อมูลความน่าจะเป็น (Damage Probability)

### 2.1 ไฟล์ `ข้อมูลโอกาสเกิดความเสียหาย (1).xlsx`
ไฟล์นี้ใช้เก็บเฉพาะข้อมูล **ความน่าจะเป็นที่จะชำรุด (Damage Probability)** โดยเปรียบเสมือนฐานข้อมูลตัวคูณที่ใช้คำนวณโอกาสที่สินทรัพย์จะพัง

| ชื่อคอลัมน์ใน Excel | ตัวแปรใน Code (Variable) | ความหมาย/คำอธิบาย |
|--------------------|-------------------------|-------------------|
| `รายการ` | `key` / `workload_item` | ชื่อรายการภาระงาน (ต้องตรงกับข้อมูลปริมาณงาน เช่น "ศาลาทางหลวง") |
| `ความน่าจะเป็นที่จะชำรุด (%)` | `damage_probability` | เปอร์เซ็นต์หรืออัตราส่วนที่จะเกิดการชำรุด (แปลงเป็นจุดทศนิยมในโค้ด เช่น 0.10) |

## 3. การแสดงผลบนหน้าบ้าน (Frontend Parameter Grid)
ตาราง Parameter Grid ฝั่ง Frontend (`currentConfig` state ใน React และใน Pydantic Schema `custom_config`) จะเก็บข้อมูลที่ผสานรวมระหว่าง "รายการงาน" กับ "ค่าตัวคูณ" เข้าด้วยกัน

| คีย์ใน JSON Payload (API) | ที่มาของข้อมูล |
|--------------------------|--------------|
| `workload_item` | มาจากตัวแปร `item` ใน `src/config.py` หรือคอลัมน์ใน `group_data_final.xlsx` |
| `category` | หมวดหมู่ (Pavement, Asset, Policy) มาจาก `src/config.py` |
| `quantity_col` | ชื่อคอลัมน์ใน DataFrame ที่ได้จากการ Merge ข้อมูล |
| `damage_probability` | ดึงมาจาก `ข้อมูลโอกาสเกิดความเสียหาย (1).xlsx` หรือ ผู้ใช้แก้ไขสดในตาราง |
| `unit_cost` | ดึงมาจาก `group_data_final.xlsx` (หรือค่าเริ่มต้นใน `src/config.py`) หรือ ผู้ใช้แก้ไขสดในตาราง |
| `apply_damage_probability` | Flag เพื่อบอกระบบว่าหมวดหมู่นี้ให้คูณความน่าจะเป็นไหม (เช่น Pavement จะเป็น False เสมอ) |

การทำความเข้าใจเอกสารนี้จะช่วยให้ทีม Developer สามารถเพิ่มคอลัมน์ใน Excel และอัปเดต Mapping Logic ในไฟล์ `src/data_loader.py` และ `src/config.py` ได้อย่างถูกต้องและรวดเร็ว
