# Data Merge Logic (`master.xlsx`)

เอกสารนี้อธิบายถึงกระบวนการดั้งเดิมที่ใช้ในการนำไฟล์ย่อยหลายๆ ไฟล์มารวมกันเป็น `master.xlsx` ที่ใช้คำนวณงบประมาณในปัจจุบัน หากในอนาคตมีการอัปเดตข้อมูลจากแหล่งต้นทาง (เช่น ได้ไฟล์ Excel ใหม่ของปีถัดไป) สามารถอ้างอิง Logic ในการดึงและรวมข้อมูลได้ตามนี้

## 1. ข้อมูลโครงสร้างพื้นฐานและปริมาณงาน (Base Data & Asset Quantities)
**แหล่งที่มา:** `group_data_final.xlsx`

*   **Sheet `group_data_final`**: เป็นข้อมูลหลัก (Base) 
    *   รหัสแขวง (`depot_code`) จะถูกแปลงเป็นตัวเลขเพื่อใช้เป็น Key หลัก (`dept3`)
    *   ดึงชื่อสังกัด (`division_name`) และชื่อแขวง (`district_name`) 
*   **Sheet `ASSET`**: เป็นปริมาณงานพวกจำนวนชิ้น (Count)
    *   ดึงจากคอลัมน์ต่างๆ เช่น "ท่อลอดทาง", "ท่อข้างทาง", "ไฟสัญญาณจราจร" ฯลฯ
    *   แปลงช่องที่เป็น "-" ให้เป็น `0`
    *   นำมา Merge เข้ากับ Base ด้วยรหัสแขวง `dept3`
*   **Sheet `input`**: ข้อมูลปริมาณงานส่วนเพิ่มเติม
    *   หากมีคอลัมน์ไหนเป็นจำนวนชิ้นให้หาผลรวม (`sum`) ตามแต่ละรหัสแขวง
    *   หากมีคอลัมน์สถิติ เช่น รถยนต์ (`veh_total`), ค่าความเรียบ (`avg_iri`), ความชัน/ความสูง ให้หาค่าเฉลี่ย (`mean`)
    *   ผลลัพธ์จะถูกนำมาเชื่อมกัน โดยใส่ prefix คำว่า `input_` เพื่อแยกว่ามาจากชีต input

## 2. ข้อมูลระยะทางดำเนินการ (Operating Distances)
**แหล่งที่มา:** `operating_distances.xlsx`
*   ดึงข้อมูล `total_distance_km` โดย Group by รหัสตัวเลข 3 หลักแรกของแขวง แล้วทำผลรวม (Sum)
*   นำมา Merge โดยตั้งชื่อคอลัมน์ใหม่ว่า `operating_distance`

## 3. ข้อมูล Cluster
**แหล่งที่มา:** `cluster_district.xlsx`
*   เชื่อมโยงข้อมูล `Cluster` ที่แบ่งกลุ่มความเจริญ (เช่น Cluster 1, 2, 3) ตามรหัส `dept3`

## 4. ข้อมูลสรุป RMMS (ราคากลางเดิม)
**แหล่งที่มา:** `แผนงาน RMMS.xlsx`
*   คัดกรองเฉพาะปีงบประมาณที่ต้องการ (เช่น ปี `2568`)
*   จัดกลุ่มตาม `dept3` (แปลงจาก `deptcode`) แล้วหาผลรวมของ:
    *   รวมค่าเช่าเครื่องจักร (`machine_rental_cost`)
    *   รวมงบประมาณทั้งหมด (`rmms_total_budget`)
    *   ผลรวมปริมาณงาน (`rmms_workload_quantity`)

## 5. การประกอบร่าง (Merge Flow)
1. นำข้อมูลจากชีต `group_data_final` มาเป็นตัวตั้ง
2. `LEFT JOIN` ข้อมูลจากชีต `ASSET` ด้วยคีย์ `dept3`
3. `LEFT JOIN` ข้อมูลสถิติจากชีต `input` ด้วยคีย์ `dept3`
4. `LEFT JOIN` ข้อมูล `Cluster` ด้วยคีย์ `dept3`
5. `LEFT JOIN` สรุปค่าใช้จ่ายเครื่องจักรและงบรวมจาก `แผนงาน RMMS` ด้วยคีย์ `dept3`
6. `LEFT JOIN` ระยะทางปฏิบัติงาน `operating_distances` ด้วยคีย์ `dept3`

7. ลบคอลัมน์ที่ซ้ำซ้อนทิ้ง เช่น `asset_district_name` และเอาคอลัมน์ `input_...` มาแทนที่ตัวแปรฝั่งสถิติที่มาจาก Base (เช่น ใช้ `input_veh_total` แทน `veh_total`)
8. เติม `0` ในช่องว่างที่มีค่าเป็น NaN สำหรับคอลัมน์ตัวเลขทั้งหมด

ผลลัพธ์สุดท้ายคือข้อมูลดิบที่พร้อมสำหรับคำนวณงบประมาณของแต่ละแขวงไว้ครบถ้วนในบรรทัดเดียว!

### ระบบปัจจุบัน (อัปเดตล่าสุด)
เนื่องจากความต้องการให้สามารถแก้ข้อมูลสถิติที่ต้นทาง (เช่น เพิ่มคอลัมน์ใน `group_data_final.xlsx`) ระบบจึงถูกปรับแก้ให้กลับไปทำ **Dynamic Merge** เมื่อตอนระบบเริ่มทำงาน (Startup) 
- ฟังก์ชัน `build_master` ใน `src/data_loader.py` จะทำหน้าที่ไปอ่านข้อมูลจากไฟล์ย่อยๆ ทุกไฟล์ตามกระบวนการข้างต้นมารวมกัน 
- หลังจากรวมเสร็จ ระบบจะสร้างและบันทึกทับไฟล์ `data/master_final.xlsx` ให้โดยอัตโนมัติ 
- **หมายเหตุ:** โปรดปิดไฟล์ `master_final.xlsx` ใน Excel ทุกครั้งที่มีการรันระบบ เพื่อป้องกันข้อผิดพลาด Permission Denied
- ไฟล์ `master_final.xlsx` เป็นเพียงไฟล์ผลลัพธ์ (Generated File) เพื่อให้ผู้พัฒนาดูภาพรวมข้อมูล หากต้องการแก้ไขสถิติให้แก้ไขที่ไฟล์ต้นทาง
