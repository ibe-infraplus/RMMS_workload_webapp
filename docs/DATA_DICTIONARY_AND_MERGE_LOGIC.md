# Data Dictionary & Merge Logic (คู่มือข้อมูลและตรรกะการรวมไฟล์)

เอกสารฉบับนี้อธิบายที่มาของตัวแปรต่างๆ (Variables) ในระบบคำนวณงบประมาณ Workload Cost ว่ามีการดึงข้อมูลมาจากไฟล์ Excel ไหน ชีตอะไรบ้าง และมีกระบวนการประกอบร่าง (Merge) ข้อมูลย่อยเหล่านั้นเข้าด้วยกันอย่างไร เพื่อสร้างเป็น `master_final.xlsx` สำหรับใช้คำนวณงบประมาณในปัจจุบัน

ไฟล์ข้อมูลดิบ (Raw Data) ทั้งหมดจะถูกเก็บไว้ที่โฟลเดอร์ `data/` ภายในโปรเจกต์

---

## 1. พจนานุกรมข้อมูลและแหล่งที่มา (Data Dictionary & Sources)

### 1.1 ไฟล์ `group_data_final.xlsx` (ข้อมูลหลักและปริมาณงานตั้งต้น)
ไฟล์นี้เป็นศูนย์กลางที่เก็บข้อมูลโครงสร้างพื้นฐาน ปริมาณงาน Asset และ Policy ของแต่ละแขวงทางหลวง

| ชีตใน Excel (Sheet) | ความหมายและตัวแปรหลัก |
|---------------------|----------------------|
| **group_data_final** | เก็บข้อมูลโครงสร้างพื้นฐาน เช่น `dept3` (รหัสแขวง 3 หลัก), `division_name`, `district_name`, `length_to2` (ระยะทางแบบ 2 ช่องจราจร) ฯลฯ สำหรับใช้คำนวณ Condition Score และ Pavement Workload |
| **group_data_final (เพิ่มเติม)** | คอลัมน์ `bridge_m` เป็นคอลัมน์ใหม่ที่เพิ่มเข้ามาเพื่อเก็บความยาวของสะพานข้ามคลองในหน่วยเมตร สำหรับใช้คำนวณงบประมาณสะพานโดยเฉพาะ |
| **group_asset** / **ASSET** | เก็บปริมาณงานหมวดสินทรัพย์แบบนับชิ้น เช่น `ท่อลอดทาง`, `สะพานลอยคนเดินข้าม` (จำนวนแห่ง) โดยจะนำไปแปลงเป็นคอลัมน์ชื่อภาษาอังกฤษเช่น `asset_pavilion_num` |
| **input** | ข้อมูลเสริมเช่น `veh_total` (AADT รวม), ค่าเฉลี่ย IRI, ความชัน ซึ่งโหลดเข้ามาเป็น Prefix `input_` เพื่อนำไปใช้คำนวณสถิติ |
| **ราคาต้นทุน / Unit Cost** | เก็บข้อมูลราคากลาง (Unit Cost) สำหรับนำไปใช้คำนวณงบประมาณในรายการต่างๆ |

### 1.2 ไฟล์ `operating_distances.xlsx` (ระยะทางดำเนินการ)
ใช้เก็บข้อมูลระยะทางดำเนินการรวมของแต่ละแขวง (เพื่อนำมาอ้างอิงและตรวจสอบ ไม่ได้ผูกกับ Unit Cost โดยตรง) 

| ตัวแปรใน Code | ชื่อคอลัมน์ใน Excel | ความหมาย/คำอธิบาย |
|--------------|--------------------|-------------------|
| `operating_distance` | `total_distance_km` | ระยะทางดำเนินการทั้งหมดในเขตพื้นที่รับผิดชอบของแขวง (กม.) |

### 1.3 ไฟล์ `ข้อมูลโอกาสเกิดความเสียหาย (1).xlsx` (ตัวคูณโอกาสชำรุด)
ไฟล์นี้เปรียบเสมือนฐานข้อมูลตัวคูณที่ใช้คำนวณโอกาสที่สินทรัพย์จะพัง (Damage Probability)

| ชื่อคอลัมน์ใน Excel | ตัวแปรใน Code | ความหมาย/คำอธิบาย |
|--------------------|--------------|-------------------|
| `รายการ` | `key` / `workload_item` | ชื่อรายการภาระงาน (ต้องตรงกับข้อมูลปริมาณงาน เช่น "ศาลาทางหลวง") |
| `ความน่าจะเป็นที่จะชำรุด (%)` | `damage_probability` | เปอร์เซ็นต์หรืออัตราส่วนที่จะเกิดการชำรุด (แปลงเป็นจุดทศนิยมในโค้ด เช่น 0.10) |

## 2. ตรรกะการนำเข้าและประกอบร่างข้อมูล (Data Merge Logic)

ปัจจุบันระบบใช้สถาปัตยกรรมแบบ **Dynamic Merge** เมื่อตอนระบบเริ่มทำงาน (Startup) 
ฟังก์ชัน `build_master` ใน `src/data_loader.py` จะทำหน้าที่ไปอ่านข้อมูลจากไฟล์ย่อยๆ ทุกไฟล์มารวมกันอัตโนมัติ

**ขั้นตอนการประกอบร่าง (Merge Flow):**
1. นำข้อมูลจากชีต `group_data_final` มาเป็นตัวตั้ง (Base)
2. **`LEFT JOIN`** ข้อมูลปริมาณงานจากชีต `ASSET` ด้วยคีย์ `dept3` (แปลงช่องที่เป็น "-" ให้เป็น `0`)
3. **`LEFT JOIN`** ข้อมูลสถิติจากชีต `input` ด้วยคีย์ `dept3` โดยหากเป็นชิ้นให้ทำผลรวม (Sum) หากเป็นสถิติให้ทำค่าเฉลี่ย (Mean)
4. **`LEFT JOIN`** ข้อมูล `Cluster` ด้วยคีย์ `dept3`
5. **`LEFT JOIN`** ข้อมูลสรุป RMMS เช่น ค่าเช่าเครื่องจักรและงบรวม ด้วยคีย์ `dept3`
6. **`LEFT JOIN`** ระยะทางปฏิบัติงาน `operating_distances` ด้วยคีย์ `dept3`
7. **Clean up:** ลบคอลัมน์ที่ซ้ำซ้อนทิ้ง เอาคอลัมน์ `input_...` มาแทนที่ตัวแปรฝั่งสถิติที่มาจาก Base และเติม `0` ในช่องว่างที่มีค่าเป็น NaN
8. ระบบจะสร้างและบันทึกทับไฟล์ `data/master_final.xlsx` ให้โดยอัตโนมัติเพื่อเป็นไฟล์สรุปให้ Developer ตรวจสอบ

> **🚨 คำเตือน:** โปรดปิดไฟล์ `master_final.xlsx` ใน Excel ทุกครั้งที่มีการรันระบบ เพื่อป้องกันข้อผิดพลาด Permission Denied เนื่องจาก Python จะพยายามเขียนทับไฟล์นี้เสมอ

## 3. การส่งผ่านข้อมูลไปแสดงบนหน้าบ้าน (API Parameter Grid)

ตาราง Parameter Grid ฝั่ง Frontend จะรับข้อมูลที่ผสานรวมแล้วผ่าน API โดยอ้างอิงจากโครงสร้างด้านล่าง

| คีย์ใน JSON Payload (API) | ที่มาของข้อมูล |
|--------------------------|--------------|
| `workload_item` | มาจากตัวแปร `item` ใน `src/config.py` หรือคอลัมน์ใน `group_data_final.xlsx` |
| `category` | หมวดหมู่ (Pavement, Asset, Policy) มาจาก `src/config.py` |
| `quantity_col` | ชื่อคอลัมน์ใน DataFrame ที่ได้จากการประกอบร่างแล้ว (เช่น `asset_pavilion_num`) |
| `damage_probability` | ดึงมาจาก `ข้อมูลโอกาสเกิดความเสียหาย (1).xlsx`  และรองรับให้ผู้ใช้แก้ไขได้ |
| `unit_cost` | ดึงมาจาก `group_data_final.xlsx` จาก sheet `Unit Cost` และรองรับให้ผู้ใช้แก้ไขได้ |
| `apply_damage_probability` | Flag เพื่อบอกระบบว่าหมวดหมู่นี้ให้คูณความน่าจะเป็นไหม (เช่น Pavement จะเป็น False เสมอ) |
